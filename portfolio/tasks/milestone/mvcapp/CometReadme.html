B2B Liquidations & Auctions

Comet — NCO Procurement App

An industrial clearinghouse engine designed to systematically parse, liquidate, and auction internal Non-Conforming Output (NCO) materials through a multi-tenant corporate bidding matrix.

⚙️ Architecture Intent

The Comet Platform provides an asynchronous portal for secondary commercial buyers to bid on production variants flagged as Non-Conforming Output. Built across an enterprise-decoupled Clean Architecture structure, it relies on SHA256 hashed authentication, safe data layer repositories, and strict isolated controllers separating administrative import controls from real-time dynamic bidding grids.

🔒 Claims-Based Cookie Authentication Sequence

Security configurations decouple raw identities by leveraging native .NET Cookie Authentication. Upon invoking AuthenticateAsync(email, password), the repository matches SHA256 hashed arrays. Once confirmed, the system converts identity claims matrices into encrypted browser cookies: 

[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Login(UserCredentialsVM model)
{
    var user = await _userRepository.AuthenticateAsync(model.Email, model.Password);
    if (user == null) return Unauthorized();

    var claims = new List<Claim> {
        new Claim(ClaimTypes.Name, user.Email),
        new Claim(ClaimTypes.Role, user.Role.Name)
    };

    var identity = new ClaimsIdentity(claims, CookieAuthenticationDefaults.AuthenticationScheme);
    await HttpContext.SignInAsync(new ClaimsPrincipal(identity));

    return RedirectToAction("Index", "Auction");
}

To simplify heavy inventory updates, administrators can upload structured manifest workbooks. The application triggers a stream parsing framework that extracts industrial lot weights, compositional qualities, and target reserve pricing tiers:

  • ProductImportService.cs: Iterates entries, filters missing parameters, converts models into system database schema types, and outputs comprehensive ImportResultDto summary statistics back to the admin window.

⚖️ Real-Time Auction & Concurrency Safeguard Framework

The AuctionService evaluates offer metrics against strict transactional constraints before recording a bid. This ensures complete audit compliance and locks product structures during active proposal checks: 

public async Task<bool> ProcessOfferAsync(int buyerId, int productId, decimal offerAmount)
{
    var product = await _productRepository.GetByIdAsync(productId);
    if (product == null || product.IsAuctionClosed || DateTime.UtcNow > product.EndDate) 
        return false;

    if (offerAmount <= product.CurrentHighestBid) 
        return false; // Blocks invalid lower bids

    var bidLog = new Bid {
        BuyerId = buyerId,
        ProductId = productId,
        Amount = offerAmount,
        Timestamp = DateTime.UtcNow
    };

    await _productRepository.UpdateHighestBidAsync(productId, offerAmount, bidLog);
    return true;
}
