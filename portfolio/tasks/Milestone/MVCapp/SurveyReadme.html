📂 System Code Map & Component Intent

Detailed decomposition of physical layers defining technical responsibilities and runtime operational boundaries.

1. Domain Layer

GlasAnketa.Domain/Models
Entity Component Technical Scope & Analytical Intent
User.csRepresents authenticated records; stores metadata identifiers (Company, Role, OUs) alongside demographics (Age, Experience).
QuestionForm.csModular survey template container grouping array nodes; contains IsActive evaluation controls.
Question.csDefines target query inputs; dictates required validation patterns and maps to layout settings.
QuestionType.csSystem lookup keys defining variations like linear Scale grids (1-10) or open multi-line Text commentary blocks.
Answer.csCaptures responses joined directly to demographics while entirely isolating individual user IDs to enforce total submission privacy.
AnswerSummary.csUnbound computation transfer objects parsing averages, percentages, and string vectors for metrics processing.
Role.csEnforces role-based permissions matrix segments (Administrator, Employee, Manager).

2. Data Access Layer

GlasAnketa.DataAccess
  • AppDbContext.cs: Houses SQL transaction contexts, handles model schema keys, and forces DeleteBehavior.NoAction overrides to safeguard data integrity.
  • DataSeedExtensions.cs: Direct database seeding interface injectors mapping core configurations and identity configurations during migrations.
  • Repository.cs: Implements decoupled generic async patterns (GetActiveAsync, GetAllAsync, AddAsync).
  • AnswerRepository.cs: Orchestrates bulk data inputs and handles cross-joining queries for group averages.
  • QuestionFormRepository.cs: Manages complex relational loading operations, multi-form loops, and state changes.

3. Business Logic Service Layer

GlasAnketa.Services
  • InjectionExtensions.cs: IoC injection root mapping operational boundaries (Scoped/Transient) cleanly into the .NET runtime core.
  • AnswerService.cs: Evaluates transactional constraints and models before allowing entries to push down to standard repositories.
  • ReportService.cs: Executes high-performance non-blocking query routines processing arrays (Age buckets, Experience ranges, and structural OU grids).
  • QuestionFormService.cs / QuestionService.cs: Validates states; explicitly blocks modifications or removals if a survey form is live in production.
  • UserService.cs: Governs authentication controls, cryptographic verification patterns, and account metadata adjustments.

4. Controller & Presentation Layer

GlasAnketa.Controllers
  • AccountController.cs: Handles form authentication targets, maps session parameters, and assigns custom navigation roots based on role scopes.
  • QuestionnaireController.cs: Checks operational request tokens, serves structural forms, and maps view state targets for employees.
  • AnswerController.cs: Intercepts complex array posts via HTTP POST and forwards model maps to backend business orchestrators.
  • AdminController.cs: Secure dashboard panel handling structure operations and user registries.
  • ReportsController.cs: Feeds management charts asynchronously and manages secure flat-file document download engines.

🔄 End-to-End Application Lifecycle Flow

1. Startup Runtime Pipeline (Program.cs)

Initializes configuration blocks, caching services, dynamic object profiles, and mounts sequential middleware rules:

HTTPS Redirect → Static Assets → Routing Engine → Authentication → Authorization Filter → Session Context
2. Session Setup & Authorization Injection

Verifies identities via secure tokens via [ValidateAntiForgeryToken] and assigns state metadata values directly into the server state matrix:

HttpContext.Session.SetInt32("UserId", user.Id);
HttpContext.Session.SetString("UserRole", user.Role.Name);
HttpContext.Session.SetInt32("CompanyId", user.CompanyId);
3. Transactional ACID Data Injection Protocol

Wraps submissions in isolated transactional contexts. To prevent duplicate submissions during sequential forms, the data engine clears historical traces matching the Active User and Form ID values before committing the final record payloads:

DELETE FROM Answers 
WHERE UserId = @userId 
  AND QuestionId IN (SELECT Id FROM Questions WHERE QuestionFormId = @formId)
